# Preferences
# Voice, tone, and style choices only.
# Formatting → templates/. Process → skills/.

# Structure:
#   Expression — mechanically measurable. Handled in Phase 1 or Phase 3 inner loop.
#   Content    — requires judgment. Handled in Phase 4 outer loop (personas).

## Expression

### Phase 1 — Batch Fix Targets
*Phase 1 finds violations and batch-fixes them. No separate scoring.*

- **Drop pronouns:** Remove "I" from summary and bullets. Use implied subject.
- **Summary length:** 3–4 sentences, ≤60 words.
- **Commodity tools:** Remove generic tools from Tools line (e.g., Google Sheets, Slack).
- **Word substitutions:** Add your preferred verb replacements here.
- **Consulting prefix:** `**Consulting (Client):**` for external client projects.
- **Work authorization:** Do not include in resume.

### Phase 3 — Scored Items
*Measured as preference expression score (10 pts). See validate/SKILL.md for breakdown.*

- **Summary closing:** Last sentence in candidate voice. (2 pts)
- **Tagline pattern:** `[Identity] | [Domain] · [Differentiator]`. (2 pts)
- **Skills label brevity:** Shortest noun for category labels. (2 pts)
- **Skills label JD alignment:** Match JD's dominant terms. (1 pt)
- **Verb echo prevention:** No repeated lead verbs in adjacent roles. (2 pts)
- **Tool names in bullets:** Only JD-specified tools in bullets. (1 pt)
